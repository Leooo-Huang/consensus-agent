# 企业 AI Agent 黑客松 — 赛题简介

## 一、企业 X 背景概况

### 基础数据

| 指标 | 数值 |
| --- | --- |
| 全球门店 | 5,000+ |
| 员工规模 | 30,000 |
| 核心市场 | 5 个(中国 / 日本 / 韩国 / 东南亚 / 美国) |

### 行业与业务定位

- **行业**:零售 · 批发制造
- **业务范围**:珠宝全产业链

### 业务链条

```
产品设计 → 供应链管理 → 品牌营销 → 终端零售
```

- 线上平台 + 线下门店,双渠道并行
- 产品、供应链、市场、运营、数字化多职能协同

### 组织结构

- **三层架构**:总部决策 + 区域管理 + 门店执行
- **一线人员**:以本地零售运营人员为主体
- **跨文化协作**:部分岗位引入具国际背景人才

## 二、赛题方向 B:协作沟通 Agent

### 企业场景

电商运营与活动筹备中,**主推商品及参与品类决策**涉及运营、商品、市场等多角色,缺乏统一决策表达方式,同一建议往往经多轮反复确认,沟通成本高。

### 赛题任务

设计**协作沟通 Agent**,根据选品提案文本(包含选品理由、市场观察、业务假设等):

1. 从不同业务角色视角进行**结构化解读**
2. 识别各角色可能的**关注点**和**潜在分歧**
3. 帮助团队形成**结构化讨论框架**

> Agent 需基于业务常识和角色特征推理各角色可能的关注点。

### 核心痛点

| 痛点 | 场景描述 |
| --- | --- |
| 跨角色视角不一致 | 不同部门关注重点各异,建议易被推翻 |
| 共识形成效率低 | 依赖会议沟通,难以快速聚焦分歧 |
| 信息表达缺乏结构 | 以经验性描述为主,缺统一框架难复用 |

### 示例

**输入**:「建议主推 A 款项链,因为小红书声量高」

**Agent 多视角输出**:

| 角色 | 关注点 |
| --- | --- |
| 市场角色 | 品牌定位是否匹配 |
| 财务角色 | ROI 预测 |
| 运营角色 | 库存与活动节奏是否匹配 |

### 创新方向

参赛者可从以下角度进行创新设计,探索如何通过 AI 提升团队沟通效率与决策一致性:

- 跨角色视角建模
- 决策表达结构
- 分歧识别
- 协作提示

## 三、评审标准

| 维度 | 权重 | 说明 |
| --- | --- | --- |
| **业务落地潜力** | 25% | 解决真实痛点、可落地、有价值 |
| **技术实现** | 20% | 代码质量、架构设计、系统可运行性 |
| **风险安全机制** | 20% | 异常监测、降级策略、人工接管、回滚机制 |
| **创新性** | 15% | 方案切入点新颖,具备独特性 |
| **演示与表达** | 10% | 路演清晰、重点突出 |
| **非技术可理解度** | 10% | 非技术人员易于理解 |

### 权重启示

- **业务 + 技术 + 风险 三大主轴占 65%** — 不能只炫技,必须证明可落地且安全可控
- **风险安全机制占比高达 20%** — 异常监测、降级、人工接管、回滚需在 Demo 中明确呈现
- **非技术可理解度 10%** — 演示必须让业务人员看得懂,避免纯技术叙事
