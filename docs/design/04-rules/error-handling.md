# 出错的时候怎么办

> 跨页面通用的错误处理原则。单个页面的具体处理写在 02-pages/P{n}-*.md 的"会出什么意外"里。

## 错误分类

| 错误类型 | 例子 | 给用户看什么 |
| --- | --- | --- |
| 网络问题 | 断网、慢、超时 | "网络不稳,3 秒后自动重试" + 重试按钮 |
| LLM Provider 故障 | Opus 5xx / 配额耗尽 / 超时 | **不显示错误**,自动降级 + Provider 角标变色 + toast 提示降级原因 |
| LLM 完全不可用(所有 Provider 都失败) | 全链路 fallback 失败 | "AI 暂时不可用,已切换到离线规则模式(仅能解构提案,不做多角色推理)" |
| 数据库异常 | Neon 连接失败 | "保存失败,你的内容已暂存浏览器,联网后会自动同步" |
| Schema 校验失败(LLM 返回不符合 Zod) | citation 缺失 / 置信度超界 | **不渲染错误内容** + LangGraph 自动重试该节点 |
| 数据为空(推理产出 0 条结论) | LLM 返回空 | "AI 未能从提案中提取足够内容,建议补充细节后重试" + 引导改提案 |
| 权限不足(V2) | 评审人想看非自己负责的提案 | "你没权限查看此内容" + 返回历史页 |
| 系统未知错误 | 任何未分类的 500 | "出了点意外,我们已记录,请稍后重试" + Sentry 上报 |

## 通用原则

### 1. 永远不展示堆栈或报错码给用户

- 给用户看人话(如"网络不稳"),技术细节进 Sentry / 审计日志

### 2. 所有可恢复错误必须有"重试 / 返回"路径

- 重试 = 同一操作再试一次
- 返回 = 退回上一个稳定状态

### 3. 关键链路(LLM 推理)必须有降级

- 降级链:Opus 4.7 → Sonnet 4.6 → Haiku 4.5 → 离线规则
- 任何降级**必须可见**(Provider 角标 + toast,见 P03 / P07)
- **禁止静默 fallback**(全局规则)

### 4. 写操作失败时,本地数据不丢

- 提案草稿 → localStorage
- 决议表单 → 失败时不关闭 + 保留输入
- Persona 编辑 → 失败时不关闭 Drawer

### 5. 读操作失败时,降级展示

- 数据库读不到 → 显示空 state + 重试按钮,不显示报错
- 部分字段缺失 → 渲染已有字段,缺的位用占位符

### 6. AI 输出不可信时,默认偏向保守

- 置信度 < 0.6 → 标红"低置信",在 UI 显示"建议人工复核",**不引导用户基于此决策**
- citation 缺失 → 该条结论直接不渲染
- LLM 输出超长 → R11 schema 校验失败 → LangGraph 重试

## 错误上报

- 前端错误 → Sentry(浏览器 SDK)
- 后端错误 → Sentry + 审计日志(双写)
- LLM 降级事件 → 审计日志(便于后续分析降级模式)
