# 这个产品是什么

> 🔥 **v2 升级(2026-05-23)**:本文档已按 [product-direction-v2.md](product-direction-v2.md) 重写。旧版"多 Agent 视角解读工具"定位已废弃。

## 一句话

**议见(YiJian)是一个基于业务方法论的企业 AI 共识形成系统。** 它把一份业务提案拆成"角色、事实、分歧、权重、证据链",输出**可复现、可解释、有方法论支撑**的决策结论,把跨部门"各说各话、反复扯皮"变成一次性可执行决策。

## 给谁用

- **电商运营经理(主提案人)**:大促/上新前要快速获得多部门共识,现在被微信群扯皮和会议拖死。
- **7 角色评审人**(运营/商品/市场/财务/品牌/供应链/**区域管理**):各自被本部门 KPI 绑架,跨部门提案看不出与自己的关系,但又必须表态。
- **决策者(总监/总经理)**:每天 10+ 决策,要一张图看出焦点和风险,不愿读 5 份评审长文。
- **用户也可作为 stakeholder 加入**:某些场景下,主提案人或决策者想"亲自下场发表观点",而不只是被动看 AI 输出。

## 做什么(v2 第一版)

### 1️⃣ 结构化:把模糊提案变成可对齐的颗粒度

- ✅ 提案文本 → 自动拆"主张 / 依据 / 假设 / 数据缺失"
- ✅ **自动识别决策类型**(选品 / 营销 / 预算 / 经营)→ 用户可手动覆盖

### 2️⃣ 目标对齐(L1 战略共识)

- ✅ 推理前先声明本次提案服从哪个公司级目标(从 OKR / Hoshin Kanri 模板选)
- ✅ 所有角色推理必须可追溯到该目标

### 3️⃣ 证据驱动(L2 事实共识)

- ✅ **证据/数据引擎**:接入内外部数据(P0 仅模拟接入:Demo 内置 4 个场景的 fixture 数据,V2 真接飞书/ERP)
- ✅ **证据链卡片**:点观点 → 看引用的数据/文档/历史案例
- ✅ 每个 Agent 的观点必须有 citation,无 citation 不渲染

### 4️⃣ 角色推理(L3 利益共识)

- ✅ 7 角色 Persona,每个角色有"业务目标 / 关键 KPI / 利益边界 / 天然冲突点 / 决策口头禅"
- ✅ **Blind First-Vote**:每个角色独立先发表,不互相看(防 anchoring bias)
- ✅ 二轮调整阶段角色可看到其他角色观点后微调
- ✅ **4 档量化**:支持 / 谨慎支持 / 反对 / 信息不足

### 5️⃣ 动态权重(L4 权责共识)

- ✅ 不同决策类型,角色权重不同:
  - 选品决策 → 供应链 / 运营 / 商品权重高
  - 营销决策 → 市场 / 品牌权重高
  - 预算决策 → 财务权重高
- ✅ 用户可手动覆盖默认权重

### 6️⃣ 共识收敛 + 反跟风算法

- ✅ **TWS(自研轨迹加权评分) 共识评分**(基于辩论轨迹评分,非简单 majority vote)
- ✅ **Premortem**:出结论前,集体预想"6 个月后失败的最可能原因"
- ✅ **稳定复现测试**:同一提案运行 3 次(Demo)/ N 次(生产),输出主要结论的一致性指标

### 7️⃣ 决策报告(7 部分)

- ✅ 结论(通过/暂缓/驳回/需补数据)
- ✅ 评分(加权总分 + 4 档支持率分布)
- ✅ 关键分歧(top 3,带角色立场对照)
- ✅ 证据链(每条结论引用的数据来源)
- ✅ 风险(Premortem 产出)
- ✅ 建议行动(RACI/DACI 责任分配)
- ✅ 纪要/共识文本(可直接发飞书/微信)

### 8️⃣ 风险护栏(同 v1,继续)

- ✅ LLM 降级链 + 数据脱敏 diff + 置信度 + 回滚 + HITL + 审计日志
- ✅ **新增"稳定性测试"面板**(同问题 3 次运行结果对比)

### 9️⃣ Demo 模拟器 + 评审导览(同 v1)

- ✅ 4 个珠宝零售选品场景(七夕/520/季度主推/跨境新品)
- ✅ URL 隐藏的评审视角对照页(`/judge-view`)

### 🔟 决议沉淀 + AAR(升级 v1)

- ✅ 决议录入升级为 **After Action Review** 模板:实际结果 vs 预测对照,反哺 Persona 备注 + 调整未来权重

## 不做什么(明确划界)

### 仍坚持的"不做"(v1 沿用)

- ❌ 实时多人共编(Yjs)— 放 V2
- ❌ 语音/视频会议转录 — Zoom/Otter 已做,不抢
- ❌ 通用 AI 队友 — 不抢 Asana/Dust

### v2 新增"不做"

- ❌ **不做总部-区域-门店全链路决策** — 链路太长,频率不够,复杂度失控;只在选品场景**轻量体现"区域管理"角色**
- ❌ **不做普通多 Agent 辩论室** — 纯辩论容易随机 / 被强势带偏 / 互相迁就
- ❌ **不做角色扮演聊天** — 不追求对话像真人
- ❌ **P0 不真接飞书/ERP** — Demo 用 fixture 模拟,V2 才真接

## 怎么算成功

### v2 升级的可观测指标

**业务侧**:
- 同一提案决策周期从 3-5 天 + N 次会议 → 1 次会议拍板
- 会议时长缩短 ≥ 30%

**产品侧**:
- Demo 4 场景 100% 跑通,粘贴 → 决策报告 **P50 ≤ 67 秒 / P99 ≤ 90 秒**(详见 [consensus-algorithm.md § 5 性能预算表](../03-tech-direction/consensus-algorithm.md))
- Demo 期间主动触发:LLM 降级 + HITL 接管 + **稳定性测试(预录视频 / 同问题 3 次运行,主要结论一致率 ≥ 67%)** + 版本回滚,各至少 1 次且 UI 全程可见
- 决策报告 7 部分**全部非空**

**评审侧**:
- 6 项打分维度每项都有 1-2 个具体产品功能直接对应(P08 评审视角对照页可一眼看到)
- **v2 增量证据**:四层共识方法论 + TWS 轨迹加权评分算法(自研,启发自 MAD) + 稳定复现测试在产品里**真实可演**,而非仅 PPT 提及
