## v2 新增子功能:AAR 模板

借鉴美军 After Action Review 方法论 + Deming PDCA 循环。提交决议时除了基础信息,还要回答 4 个 AAR 问题:

**用户怎么用**
表单下方四个文本框:
1. **预期发生什么?**(决议时的预测)
2. **实际发生什么?**(决议执行后的真实结果)
3. **差距的原因是什么?**(分析)
4. **下次怎么做更好?**(学习)

**会发生什么**
- 这 4 个字段会沉淀到决策报告归档版
- 提交后,AI 自动给出"权重调整建议":
  - 如果某角色的预测显著偏差,建议下次降低该角色权重
  - 如果某角色提的风险确实发生了,建议下次提高该角色权重
- 用户可选择"采纳建议"或"忽略"

**我们的规矩**
- 4 个字段最少有 2 个非空才能提交(避免敷衍)
- 权重调整建议仅作建议,用户不点"采纳"不生效

## 决议表单

**用户怎么用**
从 P06 讨论框架页 → "录入决议"按钮 → 进入本页表单:

- **决议状态**(单选):✅ 通过 / ⏸ 暂缓 / ❌ 驳回 / 🔄 需补数据
- **决议摘要**(必填,1-2 句):"七夕活动 A 款不主推,改为 B 款,因 ROI 测算不达标"
- **关键改动列表**(可多条):
  - "原:主推 A 款 → 现:主推 B 款"
  - "原:活动节奏 7 天 → 现:5 天聚焦"
- **会议参与者**(可多选):团队成员
- **会议日期**(默认今天)

**会发生什么**
- 提交后 → 写入 `decision` 表,关联当前 proposal_id 和 analysis_version_id
- 同时根据"影响的 Persona"列表 → 追加一条到对应 Persona 的备注字段:
  ```
  [2026-05-23 决议 prop_xxx]
  状态:通过(改主推为 B 款)
  你的角色观点采纳:ROI 担忧 → 改 SKU
  ```

**会出什么意外**
- 表单提交失败 → 不丢内容,显示错误
- 选择的 Persona 已被删除 → 警告 + 让用户重选

**我们的规矩**
- 决议**不可编辑/删除**(append-only,审计要求)
- 但允许"在原决议上发新决议"(链式)

**怎么算做对了**
- [ ] 决议保存后在 P05 对应 Persona 备注可见
- [ ] 决议在 P10 历史页可查

## 选择影响的 Persona

**用户怎么用**
表单中段 **7 个角色** checkbox(运营/商品/市场/财务/品牌/供应链/**区域管理**),可多选。AI 默认勾选"在本次推理中提出反对/担忧的角色"(因为他们的观点最值得回写)。

**会发生什么**
- 勾选后 → 提交时,该 Persona 的备注字段追加本次决议
- 不勾选某角色 → 该角色 Persona 不变

**我们的规矩**
- AI 默认勾选可被用户改

**怎么算做对了**
- [ ] 默认勾选合理(基于推理结果)
- [ ] 用户改写后保存
