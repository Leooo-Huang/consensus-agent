## v2 新增子功能:L1 目标对齐

**用户怎么用**
文本框上方一个 ComboBox "本提案服从哪个公司级目标?":
- 下拉默认列出公司级目标库(P0 内置 4-6 个示范目标,如 "Q3 七夕销售额突破 X 亿" / "提升海外市场品牌认知" / "供应链产能优化")
- 选中后显示该目标的"成功指标"(从 OKR / BSC 模板)
- 用户可选"自定义",输入目标 + 理由

**会发生什么**
- 选定后,该目标会注入所有角色 Agent 的 system prompt
- AI 提前预警:"该提案与所选目标对齐度评分:0.82(高)"
- 对齐度 < 0.5 时弹 banner 确认

**我们的规矩**
- L1 是必选项,不选不能提交
- 选完后在 P12 决策报告 § 结论 中始终引用

## v2 新增子功能:决策类型自动识别

**用户怎么用**
提案输入区下方,AI 自动识别决策类型:
- 显示徽章:"识别为:**选品决策**" (置信度 0.91)
- **5 个候选 Tab**(选品 / 营销 / 预算 / 经营 / **跨境-区域**),用户可点切换覆盖(v2.1 加 cross_border 决策类型 — 详见 [methodology.md L4 权重表](../03-tech-direction/methodology.md))
- 选定后右侧立即更新"动态权重预览"

**会发生什么**
- AI 在用户停止输入 2 秒后自动识别
- 识别失败 → 默认"选品决策"(因为黑客松 Demo 主场景)

## v2 新增子功能:动态权重预览

**用户怎么用**
右侧角色卡片列表旁边,显示该决策类型下每角色的默认权重:
- 运营 ×1.2 / 商品 ×1.3 / 市场 ×1.1 / 财务 ×1.0 / 品牌 ×0.9 / 供应链 ×1.3 / 区域 ×1.0
- 用户可在每个权重旁点击微调(0.5-2.0 范围)

**会发生什么**
- 权重调整实时反映,不影响推理前的角色选择
- 推理后在 P12 § 评分 中再次显示

## 提案文本输入

**用户怎么用**
左侧大文本框 → 粘贴或键入选品提案(选品理由、市场观察、业务假设等)。右侧有"示例提案"按钮,点击插入参考模板。

**会发生什么**
- 文本长度 < 50 字 → 提交按钮置灰 + 提示"提案过短,至少 50 字"
- 文本长度 > 5000 字 → 警告"内容较长,推理可能 > 30 秒,建议精简"
- 检测到疑似敏感字段(供应商名/价格)→ 顶部黄色 banner 提示"已识别 N 处疑似敏感信息,提交后将自动脱敏(可在风险中心查看 diff)"

**会出什么意外**
- 浏览器崩溃/页面刷新 → 文本每 10 秒自动存草稿到 localStorage,重新进入恢复
- 用户从 Word 粘贴带格式 → 自动转纯文本 + 提示"已去除富文本格式"

**我们的规矩**
- 文本框最小高度 12 行,不要太小让用户压抑
- 草稿恢复必须显示来源时间("3 分钟前的草稿")

**怎么算做对了**
- [ ] 草稿 10 秒自动保存
- [ ] 长度校验实时反馈
- [ ] 敏感词检测在用户停止输入 1 秒后触发

## 选择参与角色

**用户怎么用**
文本框右侧或下方,**7 个角色 Persona** 以小卡片排列(运营 / 商品 / 市场 / 财务 / 品牌 / 供应链 / **区域管理**),默认全部选中。可单点取消或全选。布局 4×2 网格,末尾留 1 个 "+ 新增" 占位。

**会发生什么**
- 选中 < 2 个角色 → 提交按钮置灰 + 提示"至少需要 2 个角色才能产生分歧分析"
- 选中 ≥ 2 个 → 提交可点
- 卡片显示角色头像 + 名字 + 一句话目标(从 Persona 工坊读取)

**会出什么意外**
- Persona 数据库异常 → 降级到内置 7 个 hardcoded Persona,顶部 banner 提示"自定义 Persona 暂不可用,使用内置默认值"

**我们的规矩**
- 角色卡片不能用 emoji 头像 — 用 Lucide 图标(Briefcase / Package / Megaphone / Coins / Sparkles / Truck / **Globe** 对应 7 角色,详见 [ui.md §1.6 全产品图标语义锁定表](../../plans/2026-05-23-collab-agent-ui.md))
- 默认全选,因为大多数场景"多视角越多越好"

**怎么算做对了**
- [ ] **7 角色**默认全选
- [ ] 单击切换选中状态有动画反馈
- [ ] 区域管理角色在 4 个 Demo 场景中有差异化观点(详见 [P05 角色卡](P05-persona-workshop.md))

## 提交并启动推理

**用户怎么用**
满足条件后点底部"开始分析"主按钮 → 立即跳转 P03 推理流页面,后端开始执行 LangGraph 子图。

**会发生什么**
- 立即生成 proposal_id + analysis_version_id → URL 跳到 `/analysis/{id}`
- 后端启动 LangGraph 子图(R1 解构 → R8 脱敏 → R3 多角色并发 → R5 议程生成)
- 用户在 P03 看到流式过程

**会出什么意外**
- 后端 5xx → 不跳转,弹错误 toast"启动失败,请稍后重试",草稿不丢
- LLM Gateway 完全不可用 → 进入"离线规则模式",仍可看到 R1 解构(规则版)+ 不做 R3 推理,banner 明确提示"AI 不可用,仅提供结构解构"

**我们的规矩**
- 提交后不阻塞 — 立即跳页让用户看到进度
- 失败必须给清晰错误,不能静默

**怎么算做对了**
- [ ] 提交到跳转 ≤ 500ms
- [ ] 后端异常时不丢草稿
- [ ] 离线模式可演示(评审看到降级也能拿分)
