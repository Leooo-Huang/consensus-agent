## 三段式结构

**用户怎么用**
页面默认展示 3 个 section,纵向排列:

1. ✅ **已共识结论**(所有/大多数角色都支持的论点)
   - 每条:论点 + 支持角色徽章 + 一句话依据
2. ❓ **待回答问题**(分歧点 + 必须先答才能继续)
   - 每条:问题 + 哪几个角色提出 + 默认 Owner(可改)
3. ⚠️ **数据缺失项**(AI 识别的"提案没说但角色都关心"的事)
   - 每条:缺什么数据 + 哪几个角色需要 + 建议补充来源

**会发生什么**
- 加载完成 → 3 段从上到下淡入
- 每条可单击展开 → 显示完整 citation 和该论点在矩阵中的位置
- 用户编辑 → 实时保存到当前 analysis_version

**会出什么意外**
- 推理产出三段为空 → 显示空 state + "建议补充提案细节后重新推理"
- 编辑时网络断 → 本地缓存,联网后自动同步

**我们的规矩**
- 每条结论**必须**有 citation(从 P04 矩阵继承)
- 三段顺序不能调换 — 共识先看(降低焦虑),分歧后看(聚焦)

**怎么算做对了**
- [ ] 3 段都非空时正常展示
- [ ] 每条可展开看完整 citation

## 待回答清单 Owner 分配

**用户怎么用**
每条待回答问题右侧有 Avatar 下拉 → 选择团队成员(默认显示提案人)→ 系统记录"由谁负责回答 X"。

**会发生什么**
- 分配后 → 该 Owner 的入口出现"@你"红点(P1 时通知,P0 仅 UI 标记)
- Owner 回应后 → 该问题状态变"已回应",Agent 自动重算共识度
- 多人同时分配同一题 → 后写赢 + 显示历史

**会出什么意外**
- 团队成员列表加载失败 → 退化为"自由输入名字"
- Owner 长期未响应 → P10 历史页可看到"积压清单"

**我们的规矩**
- Owner **可以是 Agent 自身**(选"AI 自动回答")→ 触发额外推理用 Web/数据补充该缺失项(V2 功能,P0 仅展示按钮)
- 默认 Owner 推断:基于问题内容 + 角色 Persona 匹配(财务问题默认分给财务负责人)

**怎么算做对了**
- [ ] Owner 可选可改
- [ ] 分配状态持久化

## 角色视角分 Tab 查看

## 一键导出 Markdown / 投屏模式

**用户怎么用**
页面右上 2 个按钮:
- 📤 **导出 Markdown** → 下载 .md 文件,可贴飞书/Notion
- 📺 **投屏模式** → 进入全屏简洁视图,字号放大 2x,适合会议室投影

**会发生什么**
- 导出 → 服务端生成 md 文本 + 图片(热力图截图嵌入)→ 下载
- 投屏模式 → 隐藏导航,大字显示,键盘 ← → 翻页(每页一段)

**怎么算做对了**
- [ ] Markdown 包含完整三段 + citation
- [ ] 投屏模式可一键 Esc 退出
