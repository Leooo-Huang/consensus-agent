## 面板 7:稳定性测试(v2 新增,演示核心)

**用户怎么用**
显示:
- 本次提案的"3 次复现"按钮(Demo 时主动按)
- 3 次运行结果对比:
  - 主要结论一致性(% 一致 / 不一致)
  - top 3 分歧 Jaccard 相似度
  - 关键证据重叠率
- 综合判定:`stable`(Lucide `CheckCircle2` + `--success`)/ `partial`(Lucide `AlertTriangle` + `--warning`)/ `unstable`(Lucide `XCircle` + `--destructive`)— **禁止 emoji,详见 [ui.md §1.6 锁定表](../../plans/2026-05-23-collab-agent-ui.md)**

**会发生什么**
- 按"复现"→ 后端启动 3 次完整 pipeline(temperature 0.3/0.4/0.5,种子 42/84/126)
- 实时进度条显示 3 个 run 状态
- 完成后展示对比表 + 一致率
- 不稳定时显示"建议补更多证据后重试"

**我们的规矩**
- 每次复现真实跑 LLM,不允许缓存假装
- 默认不自动跑(成本 ×3),Demo 时主动按
- 复现版本独立存盘(不污染原版本)

**怎么算做对了**
- [ ] 真实跑 3 次推理
- [ ] 一致率算法正确(用 Jaccard / cosine 等)
- [ ] Demo 时一致率 ≥ 67%(否则需调优 prompt 重做)

## 面板 1:降级链状态

**用户怎么用**
显示:
- **当前 Provider**:大字号 + Lucide `CheckCircle2`(success 色)+ `Opus 4.7` 字样(**禁止 emoji**,详见 [ui.md §3.2 Provider 角标](../../plans/2026-05-23-collab-agent-ui.md))
- **降级链定义**:Opus 4.7 → Sonnet 4.6 → Haiku 4.5 → 离线规则
- **最近 10 次降级事件**:时间 / from→to / 原因(超时/5xx/配额/手动)
- **手动测试按钮**:"手动降级一档"(Demo 时按这个 → Sonnet 4.6 接管,UI 实时反映)

**会发生什么**
- 真实降级时 → 此面板的"降级链"动画显示"切换路径"
- 手动测试 → 立即触发,toast 反馈

**我们的规矩**
- 离线规则模式必须**真实可跑**(自研规则版本的简化推理,不是 mock)
- "手动测试"按钮在 Demo 时必演示 1 次

**怎么算做对了**
- [ ] 真实降级可见
- [ ] 手动测试可演示且可视化反馈

## 面板 2:数据脱敏 diff

**用户怎么用**
显示并排两栏:
- 左:**原始提案**(高亮检测到的敏感字段)
- 右:**脱敏后提案**(同位置显示占位符如 `[供应商_001]` / `[价格_范围_A]`)

底部:**还原映射表**(原值 ↔ 占位符,可下载,仅本会话有效)。

**会发生什么**
- 推理结束后此面板填充本次提案的脱敏 diff
- 未检测到敏感词 → 显示"本次提案未检测到敏感字段"
- LLM 兜底检测(Haiku 4.5)运行 → 显示双引擎结果对照

**我们的规矩**
- 还原映射表**仅在本会话浏览器内存**,不持久化(防意外泄露)
- 占位符命名必须语义化(如 `[供应商名称_1]` 而非 `[REDACT_1]`),便于推理时仍有上下文

**怎么算做对了**
- [ ] diff 高亮可见
- [ ] 占位符语义可读

## 面板 3:置信度统计

**用户怎么用**
显示:
- **本次推理平均置信度**:大字号(如 `0.78`)+ 颜色环(< 0.6 红 / 0.6-0.8 黄 / > 0.8 绿)
- **逐条置信度列表**:展示每条结论的 confidence + 哪个角色 + 哪条论点

**会发生什么**
- 任何 < 0.6 结论 → P06 讨论框架页对应条目自动标红
- 平均 < 0.7 → 顶部 banner"本次推理整体置信度较低,建议补充提案细节后重跑"

**怎么算做对了**
- [ ] 置信度从 schema 强制字段读取
- [ ] 低置信度统一可见

## 面板 4:版本回滚

**用户怎么用**
显示:
- **当前版本号**:v1.3(本次推理是 proposal_id 下的第几个 analysis_version)
- **版本列表**:每版的时间戳 / 触发原因(新建 / 用户重跑 / HITL 编辑) / 摘要
- **每行操作**:"查看" / "**回滚到此版**"
- 回滚二次确认 → 创建一个**新版本**(不删除当前),指向旧版的内容

**会发生什么**
- 回滚 → 创建 new version 复制旧 content + 标记"回滚自 v1.2"
- 当前页(P04 / P06)自动刷新到回滚后版本

**我们的规矩**
- **回滚不删数据** — 永远 append-only,可追溯
- 回滚是产品级而非 git 级(用户可见可操作)

**怎么算做对了**
- [ ] 回滚生效后 P04 / P06 内容更新
- [ ] 历史版本永不丢失

## 面板 5:HITL 接管入口

**用户怎么用**
显示:
- **当前是否有暂停中的推理**:大字号状态 + "立即接管"按钮
- **接管操作面板**:
  - 查看暂停时的 checkpoint 内容
  - 选择:批准 / 编辑后继续 / 驳回重启
- **历史接管记录**:谁在什么时候接管 / 决策 / 影响哪些结论

**会发生什么**
- 进入此 Tab 时若有暂停 → 顶部红色 banner 提示
- 接管决策 → resume LangGraph checkpointer

**我们的规矩**
- 接管者必须留下决策说明(最少 5 字)
- 5 分钟无人接管 → 自动按"批准"继续

**怎么算做对了**
- [ ] 接管真实触发后续推理
- [ ] 历史接管可查

## 面板 6:审计日志

**用户怎么用**
表格视图:**时间 / 操作人 / 操作类型 / 输入 hash / 输出 hash / 备注**。

**我们的规矩**
- 日志 append-only,**不可删**
- 输入/输出 hash 用 SHA-256 防篡改

**怎么算做对了**
