## ① 结论

**用户怎么用**
顶部最显眼位置,大字号显示:
- **状态徽章**(用 Lucide icon,不允许 emoji unicode — 详见 [ui.md §1.6 全产品图标语义锁定表](../../plans/2026-05-23-collab-agent-ui.md)):

**会发生什么**
- AI 基于共识算法自动产出,但用户可手动覆盖(必须留 reason)
- 状态变化时全局自动重算其他 6 部分

**怎么算做对了**

- [ ] 状态徽章颜色与 4 档语义一致

## ② 评分(L4 加权)

**用户怎么用**
显示:
- **加权总分**:大数字 0-100(基于 **TWS 轨迹加权评分(自研)** + L4 角色权重,详见 [consensus-algorithm.md § 对策 2](../03-tech-direction/consensus-algorithm.md))
- **4 档支持率分布**:水平堆叠条(渲染时用 Lucide icon + 对应 token 色,**禁止 emoji**,详见 [ui.md §1.6](../../plans/2026-05-23-collab-agent-ui.md)):
  - 支持:`CheckCircle2` + `--success`(绿)X%
  - 谨慎支持:`CheckCircle` 线性 + `--conditional`(青绿)Y%
  - 信息不足:`HelpCircle` + `--warning`(琥珀)Z%
  - 反对:`XCircle` + `--destructive`(红)W%
- **每角色权重表**:7 行 × 2 列(角色名 + 本次权重)
- "调整权重"按钮 → 弹窗手动覆盖默认权重(L4 用户可干预)

**会发生什么**
- 权重调整后实时重算加权总分
- 调整理由必填 ≥ 5 字
- 调整历史进入审计日志

**我们的规矩**
- 权重调整必须留痕,不能静默改

**怎么算做对了**
- [ ] 加权总分公式可解释(hover 显示计算式)
- [ ] 调整权重后实时反映

## ③ 关键分歧(top 3)

**用户怎么用**
3 个并列卡片,每张:
- 论点(原文)
- 主张方 vs 反对方 角色徽章
- "为什么有分歧":AI 一句话总结
- **"如何收敛"(v2.1 GAN-A #10 修:必须输出 Schema 结构化字段,不允许 AI 随便给一句话)**:
  - **`shared_interest`**(共同利益):两方都关心的本质目标(基于 Harvard 原则谈判"focus on interests, not positions")
  - **`objective_criterion`**(客观标准):该如何用数据/事实/历史先例判断,而非主观偏好
  - **`next_step`**(下一步具体行动):一句话可执行
- 链接到 P04 矩阵对应单元格

### "如何收敛" Schema(强约束)

```typescript
type DisagreementResolution = {
  shared_interest: string;       // ≥ 10 字,识别两方共同关心的本质目标
  objective_criterion: string;   // ≥ 10 字,可量化的判断依据
  next_step: string;             // ≥ 5 字,一句话行动项
};
```

Zod 校验失败 → LangGraph 重试该论点。防 AI 给"建议双方协调"这种废话。

**会发生什么**
- 自动按"分歧热度"(权重 × 立场差)排序取 top 3
- 用户可手动加 1 条(若有第 4 个特别重要的分歧)

**怎么算做对了**
- [ ] top 3 真实反映加权后的最大分歧
- [ ] 收敛建议非空非废话
