## 新建提案入口

**用户怎么用**
点击页面正中"新建提案"主按钮 → 跳转 P02 提案输入页。

**会发生什么**
- 已有正在进行中的草稿提案 → 提示"上次还没完成的草稿,继续 / 新建"
- 第一次进入 → 直接跳 P02

**会出什么意外**
- 后端未响应 → 按钮显示 loading,3 秒未响应给"网络较慢,请重试"
- 用户连续快速点击 → 节流,只触发一次跳转

**我们的规矩**
- 主按钮永远在视觉中心,任何其他元素不抢戏

**怎么算做对了**
- [ ] 进入首页 ≤ 1 秒看到主按钮
- [ ] 点击后 ≤ 500ms 完成跳转

## Demo 一键模拟器

**用户怎么用**
首页右上"演示模式"开关 → 打开后展开 4 个预置场景下拉:
1. 小红书声量提案(A 款项链)
2. 七夕情侣对戒(节奏冲突场景)
3. 百万旗舰款(财务 vs 品牌强冲突)
4. 跨境市场新品(中日韩跨文化冲突)

点任一场景 → 自动加载提案文本 + 预设 Persona 配置 + 预设 L1 目标 + 预设证据池 → 直接跳到 P03 推理流页面。

**4 场景的"区域管理角色差异化观点"预设(v2.1 GAN-A 必杀 #2 修)**:

> 每个 fixture 场景必须给区域管理角色预写**与其他角色冲突**的观点,证明区域不是凑数。

| 场景 | 区域管理预设观点(Round 0 初始立场) |
| --- | --- |
| **场景 1 小红书声量 A 款项链** | **谨慎支持**(渲染 Lucide `CheckCircle` + `--conditional`) — "中国/日本主战场可主推,但**东南亚消费者偏好黄色金而非项链款式**,小红书声量在东南亚 KOL 影响力 < 20%,建议东南亚不主推" |
| **场景 2 七夕情侣对戒** | **反对(分区域)**(渲染 Lucide `XCircle` + `--destructive`) — "**东南亚情侣节是 2/14 而非七夕**,日本七夕(7/7)消费氛围远弱于中国,**强行同步发起=区域 ROAS 失败**;建议中国/韩国正常推,日本/东南亚换 2/14 或当地节日" |
| **场景 3 百万旗舰款** | **支持**(渲染 Lucide `CheckCircle2` + `--success`) — "**高客单价适合品牌力强的市场(中国/日本)**,可作为标杆款拉高品牌势能;东南亚/韩国本地 GDP 段位差异大,主推会拖低 conversion,可作为陈列款不强推" |
| **场景 4 跨境市场新品** | **强支持(本场景区域权重 1.5)** — "**这正是区域视角主场**:东南亚/日本市场调研显示对'轻奢镶嵌'品类需求年增 35%,建议印尼/泰国试点先于韩国;**14K vs 18K 偏好**:东南亚 14K 接受度高,日本必须 18K 起" |

> **说明(v2.1 GAN-B 修)**:本表中"谨慎支持/反对/支持"是 4 档态度名,实现时**渲染用 Lucide icon + token 色,严禁 emoji unicode**。详见 [ui.md §1.6 锁定表](../../plans/2026-05-23-collab-agent-ui.md)。

(每个场景的预设观点会被注入 fixture JSON,P03 推理时区域角色基于此发挥)

**会发生什么**
- 选场景 → 后端载入预置 fixture JSON → URL 带 `?demo=scenario-1` 参数 → 跳转 P03
- 路演现场 LLM 真实故障 → 自动切换到"录像回放模式"(预录的流式输出)
- 用户重复触发 → 直接覆盖上一次的提案 session(草稿不丢)

**会出什么意外**
- fixture JSON 文件缺失 → 显示"该场景预置数据加载失败,请选另一个或手动新建"
- 网络抖动 → 重试 1 次,仍失败给清晰错误

**我们的规矩**
- 4 个场景的 fixture 数据**必须真实跑通过**一次,不允许"看起来能用但实际跑炸"
- 路演时优先用此入口,而不是临场手输提案
- 录像回放模式必须可视化标记"当前为回放,非真实推理"(避免误导评审)

**❓ 需你拍板**
- 4 个场景的提案文本是从企业 X 拿真实数据(脱敏),还是基于公开珠宝行业知识虚构? — 选项:真实脱敏 / 虚构 — 影响:业务真实感 vs 数据合规

**怎么算做对了**
- [ ] 4 个场景均可一键加载并跑完整链路 ≤ 30 秒
- [ ] 录像回放模式可在演示前手动切换并明确标识
- [ ] 切换场景时 URL 同步更新,可分享深链
