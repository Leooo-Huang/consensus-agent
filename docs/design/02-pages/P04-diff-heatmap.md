## 主张 × 角色 热力矩阵

**用户怎么用**
页面主体一张矩阵:
- **X 轴**:从提案拆出的"原子论点"(如"主推 A 款项链" / "依据是小红书声量")
- **Y 轴**:**7 个角色 Persona**(运营 / 商品 / 市场 / 财务 / 品牌 / 供应链 / **区域管理**),每行附**动态权重**数字(如 `运营 ×1.2`)
- **单元格状态**(v2.1 4 档,用 Lucide icon + token 色,**禁止 emoji**,详见 [ui.md §1.6 锁定表](../../plans/2026-05-23-collab-agent-ui.md)):
  - **支持**:`CheckCircle2` + `--success`
  - **谨慎支持**(v2.1 新):`CheckCircle` 线性 + `--conditional`
  - **反对**:`XCircle` + `--destructive`
  - **信息不足**(v2.1 替代"无数据/中立"):`HelpCircle` + `--warning`
- **Anchoring 警示**(v2.1):被 Anchoring 检测识别为"可疑跟风"的 cell,周围加 2px 橙色边框 + Tooltip "可能被 Round 0 观点 anchor"

**会发生什么**
- 加载完成 → 矩阵从左到右逐列淡入(framer-motion stagger)
- 单击单元格 → 弹出 Drawer 展示"该角色对该论点的完整关注点 + citation"
- 论点过多(> 12)→ 横向滚动,顶部固定角色列

**会出什么意外**
- 推理失败导致部分单元格无数据 → 显示 Lucide `HelpCircle` + `--muted` 色 + Tooltip"该角色未能完成对此论点的分析"(注意与"信息不足"态度不同:此处是系统失败,色用 muted;信息不足是角色主动表态,色用 warning)
- 数据全空 → 显示空 state + 重试按钮 + 提示"AI 未能产生足够的多角色视角,建议补充提案细节"

**我们的规矩**
- 矩阵必须 SVG 渲染(截图清晰,Demo / SNS 传播友好)
- 颜色 + 图标双通道(色盲友好)
- 不能用 emoji 当 cell icon — 严格按上表 4 档 Lucide(CheckCircle2 / CheckCircle / XCircle / HelpCircle)

**怎么算做对了**
- [ ] **7 角色** × 6+ 论点矩阵渲染流畅,每行附权重数字
- [ ] cell 单击 ≤ 200ms 弹出 Drawer
- [ ] 12 论点以上横向滚动 + 固定列

## 一句话核心分歧卡片(顶部常驻)

**用户怎么用**
矩阵上方一张大卡片,大字号显示 AI 提炼的"本次最大分歧"一句话(如:"市场角色坚持 A 款符合品牌调性,但财务角色认为 ROI 测算缺失关键数据")。

**会发生什么**
- 推理完成 → AI 自动产出此句(R11 压缩护栏强制 ≤ 1 句)
- 用户编辑 → 可手动覆盖,保存到当前 analysis_version

**我们的规矩**
- 字号必须最大,占据视线焦点
- 文字不能 > 50 字,超过 LangGraph 自动重写

**怎么算做对了**
- [ ] 顶部常驻,滚动矩阵时不消失
- [ ] AI 输出长度 ≤ 50 字

## Cell hover citation popover

**用户怎么用**
鼠标 hover 任一单元格 → 浮窗显示:
- 该角色 + 该论点的态度(支持/中立/反对 + 置信度 0-1)
- 关注点 list(2-5 条)
- **每条关注点的 citation**:
  - 📄 来自提案原句:"...(高亮原文)"
  - 🎭 基于角色规则:"...(Persona 的 KPI 或风险偏好规则原文)"
- 置信度 < 0.6 显示"低置信"红色徽章

**会发生什么**
- hover ≥ 300ms 弹出 → 避免快速划过误触
- 离开 cell → 100ms 后消失
- 点击 popover 内"打开完整视图"→ 跳 P06 讨论框架页对应章节

**我们的规矩**
- citation **必须**存在,无 citation 的关注点直接拒绝渲染(从源头拒绝幻觉)
- 原文高亮使用 `<mark>` 标签 + Tailwind 黄底,可读

**怎么算做对了**
- [ ] 100% 关注点有 citation
- [ ] 低置信度自动标红
- [ ] hover 流畅无闪烁
