## 推理步骤进度条(v2.1 9 节点)

**用户怎么用**
页面顶部**两层进度条**:

**上层:四层共识语义进度**(给业务人员看的)
```
[L1 战略对齐] → [L2 事实证据] → [L3 角色立场] → [L4 权重收敛] → [报告生成]
```

**下层:9 节点详细进度**(给技术评审看的)
```
[1 结构化+决策类型] → [2 L1 目标对齐] → [3 L2 证据召回] →
[4 Round 0 Blind First-Vote 7 并发] → [5 Round 1 二轮调整(伪并发)] →
[6 TWS 轨迹加权评分] → [7 L4 权重加权] → [8 Premortem 7 并发] → [9 决策报告生成]
```

> **节点数说明**:**8 主节点 + Premortem(P0 必做) = 共 9 节点**。Premortem 因 P12 § 风险依赖,不允许可选关闭。详见 [architecture.md](../03-tech-direction/architecture.md)。

当前节点高亮 + 旋转 icon,已完成步打勾,未开始步灰显。每节点显示子标题"正在做什么"(如"正在拆解 7 个论点" / "正在召回 12 条相关证据" / "运营/财务/品牌 3 个角色已完成 Blind First-Vote")。

### 节点 4 特殊设计:Round 0 Blind First-Vote 并发可视化

这是 **v2 演示核心**之一。点开节点 4 详情面板,展示 **7 个角色并发圆点**:

```
       运营   商品   市场   财务   品牌   供应链   区域管理
       ●     ●     ✓     ●     ✓     ●         ●
       旋转  旋转  完成  旋转  完成  旋转      旋转
```

- 每角色一个圆点(对应 Lucide icon:Briefcase / Package / Megaphone / Coins / Sparkles / Truck / Globe)
- 角色推理中 → 圆点脉冲 + Loader 旋转
- 完成 → 变 ✓ + 显示该角色的态度初稿(4 档之一 + 一句话理由)
- 全部完成才推进到节点 5(Round 1)

**演示价值**:让评审亲眼看到"7 个角色独立并发,不互相看" — Blind First-Vote 真做了。

**会发生什么**
- 每节点开始 → 当前节点高亮 + 子标题显示"正在做什么"
- 每节点完成 → 打勾 + 显示耗时(节点 4 显示"7 角色并发完成,总耗时 X 秒")
- 全部完成 → 整条进度条变绿 + 显示"查看完整决策报告"主按钮(跳 P12)

**会出什么意外**
- 单节点超时 30 秒 → 标红"较慢,正在重试" + 触发 LLM 降级
- 节点 4 中某角色失败 → 该角色圆点变 ⚠ + 继续其他角色,不阻塞
- 多节点连续失败 → 顶部红色 banner"AI 不稳定,正在切换到离线规则模式"
- **L1 目标对齐度 < 0.5** → 节点 2 完成后弹 Dialog "该提案可能偏离声明目标,请确认是否继续"

**我们的规矩**
- 进度条永远不"假装进度" — 真实 LangGraph 节点完成事件才推进,不允许 setInterval 模拟
- 降级事件必须可见(toast + 顶部 banner 双重提示)
- Round 0 7 角色**真实并发**(LangGraph Send API),不允许伪并发
- **四层共识进度条是产品视觉锚点**,演示时演讲者反复指它说"我们做的是方法论框架,不是辩论室"

**怎么算做对了**
- [ ] **9 个节点**真实绑定 LangGraph 节点完成事件(数量与 architecture.md 一致)
- [ ] 节点 4 Blind First-Vote **7 角色圆点真实可视并发**
- [ ] 上下两层进度条同步推进
- [ ] 单节点耗时 ≥ 1 秒,< 1 秒不显示"完成动画"避免视觉混乱
- [ ] 端到端 P50 ≤ 90 秒(详见 [consensus-algorithm.md 性能预算表](../03-tech-direction/consensus-algorithm.md))

## 每步流式输出预览

**用户怎么用**
进度条下方一个折叠面板,展开后实时显示当前步骤的部分输出(如"R1 解构"展开会看到正在生成的"主张 1: ... 依据: ...")。

**会发生什么**
- streamObject 部分输出到达 → 增量渲染,**不闪烁**
- 用户可手动收起 → 收起后不影响后端流式继续
- 全部完成 → 折叠面板出现"查看完整 JSON"链接(给技术评审看)

**会出什么意外**
- 流式中断 → 显示已收到部分 + "连接中断,正在重试"
- 输出格式不合 Schema → 不渲染,记录到审计日志,LangGraph 自动重试该节点

**我们的规矩**
- 流式渲染不能让前面已渲染的内容跳动
- "查看完整 JSON"是技术细节,默认隐藏

**怎么算做对了**
- [ ] 流式渲染平滑无跳动
- [ ] 部分输出已可读

## 手动"暂停 / 人工接管"入口(HITL)

**用户怎么用**
进度条右侧"⏸ 暂停"按钮 → 点击后 LangGraph `interrupt()` 触发,推理暂停,弹出 HITL 面板让用户:
- **批准当前结果**继续
- **编辑当前步输出**后继续
- **驳回**整个推理重启

**会发生什么**
- 暂停触发 → 后端 checkpoint 保存,前端显示"已暂停,等待你的指令"
- 用户决策后 → checkpointer resume,继续后续步骤
- 5 分钟无人响应 → 自动按"批准"继续(避免卡死)

**会出什么意外**
- 暂停时网络断开 → checkpoint 已存盘,刷新页面后可继续(在 P10 历史页恢复)
- 多人同时暂停 → 后写赢

**我们的规矩**
- HITL 不是"必须使用",但**Demo 时必演示一次**(直接拿风险机制 20% 分)
- 暂停状态在 P07 Safety Center 也能看到 + 接管

**怎么算做对了**
- [ ] 暂停在 1 秒内生效
- [ ] checkpoint 在 Neon 中可见
