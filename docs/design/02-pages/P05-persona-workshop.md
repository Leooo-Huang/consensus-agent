### v2.1 新增:语气模式切换(GAN-A v2 必杀 #3 修)

> **背景**:GAN-A v2 指出"决策口头禅"(如财务说"投入产出算过没?")在严肃企业评审场合可能显玩具感。
>
> **解法**:Persona Drawer 顶部加 **"语气模式"开关**(Lucide `Sliders` icon),两个选项:
> - **正式模式(默认)**:角色发言用书面语,如"基于本次投资 ROI 测算,建议补充以下数据..."
> - **口头禅模式**:用 Persona 的 `decision_catchphrase`,如"投入产出算过没?数据靠不靠谱?"

**实现**:每个 Persona 在 system prompt 注入时,根据 mode 选择 `formal_prompt` 或 `casual_prompt`:
- `formal_prompt`: "你是 {role},发言用正式商务语气,基于事实和数据"
- `casual_prompt`: "你是 {role},发言用 {decision_catchphrase} 的语气,符合本部门日常口吻"

**Demo 默认正式模式**(避免被认为玩具);演讲者口播时可现场切换演示口头禅模式,展示产品灵活性。
- 角色图标不能用 emoji,必须 Lucide

- [ ] 每张卡片显示完整 KPI 列表

## 单个角色编辑 Drawer

**用户怎么用**
点卡片 → 右侧 Drawer 滑入,表单字段:
- 角色名
- 目标(一句话)
- 关键 KPI(多行,可增删)
- 风险偏好(下拉:保守 / 中性 / 激进)
- 决策口头禅(一句话)
- 备注(自由文本,会被 R3 推理引擎参考,**也是决议回写的目标字段**)

**会发生什么**
- 表单提交 → 实时 Zod 校验 → 保存到 Neon
- 保存后 → 该角色在当前 session 立即生效(下次推理用新值)
- 备注字段超长(> 2000 字)→ 截断警告

**会出什么意外**
- 保存失败 → 不关闭 Drawer,显示具体错误
- 并发编辑(团队成员同时改)→ 后写赢 + 显示最近修改人和时间

**我们的规矩**
- 备注字段是 R6 决议沉淀的回写目标,需在 UI 标注"决议回写会追加到这里"
- 编辑后,**已存在的 analysis_version 不会被反向修改**(数据不可篡改),只影响后续推理

**怎么算做对了**
- [ ] 修改后下次推理生效
- [ ] 备注字段可见"由 P09 决议页追加"的历史条目
